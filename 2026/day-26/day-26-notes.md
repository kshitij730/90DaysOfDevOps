🚀 Day 26 of #90DaysOfDevOps

Today I discovered a tool that made me question how much time I spend switching between the terminal and browser.

**GitHub CLI (`gh`)**

Before today, creating repositories, managing issues, reviewing PRs, and checking workflow runs usually meant opening GitHub in a browser.

Now I can do all of that directly from my terminal.

Today's hands-on learning:

✅ GitHub CLI Installation & Authentication
✅ Creating Repositories from Terminal
✅ Managing Issues with `gh issue`
✅ Creating & Merging Pull Requests with `gh pr`
✅ Monitoring GitHub Actions using `gh run`
✅ GitHub API Calls using `gh api`
✅ Managing Releases, Gists & Repository Search

One thing became very clear:

**The terminal is no longer just for Git. It's becoming the control center for GitHub itself.**

For DevOps engineers, reducing context switching means faster workflows, better automation, and more time focused on solving problems instead of navigating tabs.

Every day, the picture becomes a little clearer.
